## 1. テスト目的

Phase 2b-MVP（複数写真まとめて解析）の本番相当動作を確認する。

- 無料枠での 1枚 / 3枚 / 5枚 解析が実用的に機能するか
- 429 / 503 エラー発生頻度・タイミングを把握する
- 解析時間が利用者にとって許容範囲内か判断する
- 保存データに画像が混入しないことを実データで確認する
- 既存の 1枚写真解析・テキスト解析が壊れていないことを確認する

## 11. 合格基準

以下の **すべて** を満たした場合に合格とする。

| 基準 | 合格条件 |
|---|---|
| 1枚解析 | エラーなく完了し、保存される |
| 3枚解析 | 3件すべて解析完了（Rate Limit 発生時を除く） |
| 5枚解析 | 少なくとも1件が解析完了し、部分保存できる |
| 保存データ | `inputMethod: "bulk-photo"` で正しく保存される |
| 画像混入なし | `previewUrl` / `file` / `blob:` が Meal データに含まれない |
| 既存機能 | 1枚写真解析・テキスト解析・フォーム解析が正常に動作する |
| 二重保存なし | 保存ボタンを複数回押しても件数が増えない |
| コンソールエラー | アプリ起動・操作中に重大エラーが出ない |

## 12. 次の改善判断基準

テスト結果に基づき、以下の観点で次フェーズの実装を判断する。

| 観点 | しきい値 | 判断 |
|---|---|---|
| 3枚解析の合計時間 | 60秒超 | UX改善を検討（プログレス表示強化 / 待機時間調整） |
| 5枚解析での 429 発生率 | 50%超 | 「5枚は推奨しない」のUI警告追加を検討 |
| Rate Limit 後の操作性 | 「再試行できない」との声 | 個別再試行フロー改善を検討 |
| 食品名の誤認識頻度 | 毎回修正が必要 | 画質要件の明示・画像リサイズパラメータ見直しを検討 |
| 保存後の履歴確認 | 「どれが bulk-photo か分からない」 | 履歴表示に `inputMethod` バッジ追加を検討 |
